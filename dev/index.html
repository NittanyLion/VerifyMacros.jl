VerifyMacros

Documentation for VerifyMacros.jl.

Overview

This package provides convenient macros that verify whether given variables have given properties. These macros apply to types, fields, keys, properties, axes, supertypes, equality, length, size, file/directory existence, and generic conditions.

For instance,

    @verifytype x Float64

verifies that x is a Float64 and prints a convenient error message, otherwise. An example: 

julia> using VerifyMacros
julia> x = 3.03.0
julia> @verifytype x IntERROR: TypeError: x is of type Float64; was expecting a Int64
#= REPL[3]:1 =#

If desirable, one can change the description of the variable by providing a third argument, thus:

julia> using VerifyMacros
julia> x = 3.03.0
julia> @verifytype x Int "Some variable"ERROR: TypeError: Some variable is of type Float64; was expecting a Int64
#= REPL[3]:1 =#

There are multi-argument versions of the same. For instance, one can do

julia> using VerifyMacros
julia> x = 3.03.0
julia> y = "blooper""blooper"
julia> @verifytypes (x,AbstractFloat) (y,Symbol,why)ERROR: TypeError: why is of type String; was expecting a Symbol
#= REPL[4]:1 =#

VerifyMacros.@verifyaxes — Macro
@verifyaxes(array, expected_axes[, name])

Verify that array has the specified expected_axes. Throws a descriptive error if axes don't match. Optionally provide a custom name for the array in error messages.

source
VerifyMacros.@verifyaxesm — Macro
@verifyaxesm((array1, axes1), (array2, axes2), ...)

Batch verify multiple axes checks. Each argument must be a tuple of (array, expected_axes[, name]). Equivalent to multiple @verifyaxes calls.
